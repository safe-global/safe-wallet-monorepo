# Security Checks Matrix

**Origin key:**

- `Existing` = this check's logic existed in the app before this branch (shown as a warning/banner elsewhere)
- `New` = logic added in this branch
- `Planned` = not yet implemented

**Type key:**

- `Account` = on-chain property of the Safe, same for all viewers
- `User` = depends on the individual user's local state or preferences

**Status key:**

- `Clear` = Healthy, no issues (counts positively toward score)
- `Partial` = Needs attention, actionable (counts negatively toward score)
- `Issue` = At risk, should be addressed (counts negatively toward score)
- `N/A` = Not applicable to this Safe (excluded from score calculation, greyed card)
- `Inconclusive` = Unverified, check cannot be completed (excluded from score calculation, greyed card)
